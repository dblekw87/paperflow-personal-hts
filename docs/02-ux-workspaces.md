# 화면과 사용자 흐름

## 1. 정보 구조

상단 전역 바는 시장 선택, 종목 검색, KIS 연결 상태, 데이터 지연, 현재 가상 총자산을 제공한다. 왼쪽 탐색은 다음 작업공간으로 이동한다.

종목 검색은 KIS 공식 KOSPI·KOSDAQ 종목 마스터의 한글명과 단축코드를
로컬 인덱스로 조회한다. 완성형 한글 한 글자 또는 종목코드 첫 숫자부터 최대
20개를 자동완성하고 초성 단독 입력은 조회하지 않는다. `Ctrl+K`로 포커스하고
방향키·Enter 또는 마우스로 선택하면 차트·호가가 함께 있는 종목
워크스페이스로 이동한다. 마스터는 로컬에 24시간 캐시하며 오래된 캐시를
사용할 때는 상태 문구로 구분한다.

1. 시장 대시보드
2. 종목 순위/스캐너
3. 종목 워크스페이스
4. 관심종목
5. 포트폴리오
6. 주문·체결
7. 뉴스·시황
8. 설정·진단

## 2. 시장 대시보드

### 목적

현재 시장 상태와 계좌 상태를 10초 안에 파악한다.

### 구성

- KOSPI, KOSDAQ, 미국 주요 지수와 장 상태
- 상승/하락 종목 수, 시장 거래량·거래대금
- 상승률·하락률·거래대금·거래량 상위 미니 테이블
- 국내 뉴스, 미국 뉴스
- 총자산, 당일 손익, KRW/USD 현금
- REST/WebSocket 연결 배지와 마지막 수신 시각

모든 카드에는 `기준 시각`과 `live/delayed/stale` 배지가 있다. 데이터가 없을 때 0으로 표시하지 않는다.

## 3. 종목 순위/스캐너

### 탭

- 거래대금 상위
- 거래대금 증가율
- 거래량 상위
- 거래량 증가율
- 상승률
- 하락률
- 체결강도
- 시가총액
- 관심종목
- 오늘의 주도주·주도 테마

### 공통 열

종목명, 코드, 시장, 현재가, 전일 대비, 등락률, 거래량, 거래대금, 시가총액, 데이터 시각.

주도 테마 보기에는 산업→테마→하위 테마 경로, 거래대금, 전일 동시간 20일 중앙값 대비 가속도, 시장 거래대금 점유율, 상승 breadth, top1/top3 집중도, 대표 기여 종목과 `LEADING/EMERGING/ROTATING/WEAK`를 표시한다.

증가율 탭 상단에는 `전일 동시간` 또는 `전일 최종` 비교 기준을 항상 노출한다. 값이 로컬 계산이면 `계산값`, KIS 원본이면 `KIS 순위` 배지를 붙인다.

### 상호작용

- 단일 클릭: 오른쪽 미리보기 패널
- 더블 클릭/Enter: 종목 워크스페이스 열기
- 별 아이콘: 관심종목 추가/해제
- 열 헤더: 로컬 정렬
- 필터: 시장, 업종, 가격대, ETF 포함, 최소 거래대금

## 4. 종목 워크스페이스

HTS다운 밀도를 유지하되 핵심 흐름을 왼쪽에서 오른쪽으로 배치한다.

**차트와 호가창은 항상 같은 페이지·같은 종목 context로 동시에 표시한다.** 사용자는 resize할 수 있지만 둘 중 하나를 collapse하거나 다른 탭으로 옮길 수 없다. 좁은 화면에서는 주문·뉴스를 아래로 보내고 차트·호가의 동시 가시성을 우선한다.

```text
┌ 종목 헤더: 현재가 · 등락률 · 거래량 · 거래대금 · 상태 ┐
├───────────────┬────────────────────┬──────────────┤
│ 호가/실시간체결 │ 차트               │ 주문 티켓     │
│               │                    │ 보유/손익      │
├───────────────┴────────────────────┼──────────────┤
│ 종목정보 · 뉴스 · 왜 움직였나                        │
└────────────────────────────────────────────────────┘
```

### 호가

- 매도 호가는 위, 매수 호가는 아래에 가격·잔량·잔량 변화를 표시한다.
- 최우선 호가와 예상 체결가를 강조한다.
- 미국 데이터가 1호가만 제공되면 빈 깊이를 꾸며내지 않고 `1호가 제공`이라고 표시한다.

### 차트

- 1분, 5분, 15분, 30분, 60분, 일, 주 전환
- OHLC candle과 거래량·거래대금 보조 패널
- 가격·거래량·거래대금 SMA/EMA 기간 설정
- 로컬 모의 fill의 매수·매도와 부분·전량 체결 가격 마커
- 거래소 시간/KST 표시 전환
- 데이터 공백과 정규장 밖 구간을 시각적으로 구분

시장 candle은 `KIS_CANONICAL_MARKET_DATA`, 체결 마커는 `LOCAL_PAPER_FILL`로 표시한다. 주문 제출가와 체결가를 혼동하지 않으며 실제 체결이 없는 취소·거부 주문은 가격 마커로 그리지 않는다.

### 주문 티켓

- 매수/매도, 시장가/지정가, 가격, 수량
- 예상 주문금액, 최대 가능 수량, 비용 추정
- 사용 시세와 수신 시각
- 실제 주문이 아닌 `모의 주문` 고정 라벨
- 제출 뒤 `clientOrderId` 기준 단 한 번만 반영

### 왜 움직였나

- 한 줄 요약
- 확인된 수치 근거 카드
- 관련 뉴스와 시각
- 시장/업종 비교
- 반대 근거 또는 불확실성
- 신뢰도와 “추정” 라벨

### 글로벌 시황

- 금리·물가·고용·성장·환율·유동성·재정·무역·원자재·신용·정책·지정학·재난·기업·기술 사건의 한국어 제목과 원문 링크
- 경제지표 actual/consensus/prior/revision, surprise와 공식 발표시각
- category별 `AVAILABLE/UNSUPPORTED/MISSING/DELAYED/STALE` coverage
- 공식 공개, 앱 탐지, 번역 완료, 시장 반응의 타임라인
- KOSPI·KOSDAQ·USD/KRW·QQQ/USO proxy와 관련 업종 반응 strip
- 공식 확인, 가능한 전달 경로, 동시 발생, 근거 부족의 관계 수준
- 금리·실적·지수 리밸런싱 등 대안 가설과 counter-evidence

사건 marker는 차트에 선택적으로 표시하지만 매수·매도 신호처럼 표현하지 않는다. `USO`는 무료 유가 방향 ETF proxy이며 CME WTI 선물로 표시하지 않는다.

## 5. 포트폴리오

- 총자산과 통화별 현금
- 보유종목: 평균단가, 수량, 현재가, 평가액, 실현/미실현 손익
- 자산곡선과 일별 손익
- 시장·통화·종목 비중
- 가격이 stale인 종목 수와 총자산 산출 품질
- 초기자본 재설정은 새 계좌 생성 또는 명시적 조정 원장으로 처리

KRW와 USD를 하나의 숫자로 합칠 때는 적용 환율과 기준 시각을 함께 보여준다.

## 6. 주문·체결

- 주문 상태 타임라인
- 주문과 체결을 별도 행/상세 패널로 구분
- 기간, 시장, 종목, 방향, 상태 필터
- 미체결 주문 취소
- CSV 내보내기
- 거부 주문은 기계 판독 코드와 사용자 설명을 함께 표시

## 7. 뉴스·시황

- 국내/미국/관심/보유 탭
- 제목, 제공사, 발행 시각, 관련 종목, 원문 열기
- 중복 기사는 묶어서 개수를 표시
- 본문 권한이 없으면 앱 내부에서 본문을 재현하지 않는다.
- 외부 링크는 허용 도메인과 `https`를 검증한 후 OS 브라우저로 연다.

## 8. 설정·진단

- KIS 키 등록/교체/삭제와 연결 테스트
- 초기 KRW/USD, 비용·슬리피지, stale 임계값
- 현재 REST 호출량, 캐시 적중, WebSocket 구독, 재연결
- 원본 오류 코드와 사용자 친화적 설명
- DB 백업, 복원, 무결성 검사
- 로그 내보내기 전 자동 비밀정보 제거

## 9. 공통 상태 표현

| 상태          | 의미                        | UI                     |
| ------------- | --------------------------- | ---------------------- |
| `live`        | 기대 주기 안에 데이터 수신  | 녹색, 상대 지연 표시   |
| `delayed`     | 공급자가 지연 데이터로 표시 | 주황색, 지연 범위 표시 |
| `stale`       | 앱 기준 시각을 초과         | 주황/빨강, 주문 제한   |
| `offline`     | 연결 없음                   | 빨강, 마지막 정상 시각 |
| `unsupported` | 공급 범위 밖                | 회색, 대체값 없음      |

색만으로 상태를 전달하지 않고 텍스트·아이콘을 함께 쓴다.
