# 제품 요구사항

## 1. 제품 정의

개인이 국내·미국 주식과 주요 지수·선물의 실시간 시장 데이터를 보며 실제 HTS와 유사하게 모의매매를 연습하는 Windows 우선 Electron 데스크톱 앱이다. 서버 계정이나 클라우드 없이 단일 사용자의 컴퓨터에서 동작한다.

### 목표

- 실시간 체결·호가와 캔들 차트를 한 화면에서 관찰한다.
- 거래대금, 거래량, 등락률과 그 변화로 움직이는 종목을 빠르게 찾는다.
- 종목정보, 시황, 국내·미국 뉴스와 정량 근거를 함께 본다.
- 로컬 가상자산으로 주문하고 체결·포지션·손익을 복기한다.
- 데이터가 지연되거나 설명 근거가 부족할 때 이를 숨기지 않는다.

### 제품 원칙

1. `KIS = 읽기 전용 시장 데이터`, `로컬 앱 = 모의 투자 원장`으로 분리한다.
2. 실전 주문 기능은 UI에서만 숨기는 것이 아니라 네트워크 어댑터에서 구현하지 않는다.
3. 모든 금액·시세에는 통화, 시장, 원천, 거래소 시각, 수신 시각을 붙인다.
4. 오래된 데이터는 최신처럼 표시하지 않는다.
5. 설명은 근거를 보여주며 추천이나 확정적 원인으로 표현하지 않는다.

## 2. 대상 범위

### MVP

- 국내: KOSPI·KOSDAQ 일반 주식과 ETF
- 미국: NASDAQ·NYSE·AMEX 일반 주식과 ETF
- 정규장 시세와 모의 주문
- KRW·USD 별도 가상 현금
- 시장가·지정가, 매수·매도, 취소, 부분체결
- 종목 검색, 관심종목, 순위, 종목 상세, 포트폴리오, 주문·체결 내역
- KIS 제공 범위의 뉴스 제목·제공사·시각·링크
- 규칙 기반 움직임 설명
- 시장 컨텍스트용 KOSPI200 주간·KRX 야간 선물, Nasdaq 선물, 원유 선물 실시간 시세·차트

### MVP 제외

- 실제 주문, 실제 계좌 잔고 동기화, 신용·공매도
- 선물·옵션의 모의 주문·증거금·일일정산, 채권·암호화폐
- 다중 사용자, 서버 로그인, 클라우드 동기화
- 자동매매와 무인 주문
- 미국 소수점 매매
- 거래소의 정확한 큐 우선순위 재현
- 뉴스 본문 무단 수집
- 세금 신고 또는 투자 자문 용도의 계산

## 3. 주요 사용자 흐름

1. 앱을 열고 DB·REST·WebSocket 연결 상태를 확인한다.
2. 국내/미국 시장을 선택한다.
3. 상승률·하락률·거래량·거래대금·증가율 순위를 탐색한다.
4. 종목을 열어 실시간 호가·체결·차트·정보·뉴스를 본다.
5. “왜 움직였나”에서 수치와 뉴스 근거를 확인한다.
6. 주문 티켓에 방향, 유형, 가격, 수량을 입력한다.
7. 예상 금액·비용·데이터 시각을 확인하고 모의 주문을 제출한다.
8. 부분/전량 체결, 현금, 포지션, 실현·미실현 손익 변화를 본다.
9. 주문·체결 내역과 자산곡선으로 거래를 복기한다.

## 4. 기능 요구사항

### FR-01 연결과 인증

- KIS App Key/Secret은 OS 보안 저장소로 암호화한다.
- 앱 시작 시 접근 토큰과 WebSocket 접속키를 준비한다.
- REST, WebSocket, DB 상태와 마지막 정상 수신 시각을 각각 표시한다.
- WebSocket 단절 시 지수 백오프와 jitter로 재연결한다.
- KIS 장애 중에도 저장된 포트폴리오와 거래 내역은 열 수 있다.

### FR-02 실시간 시세

- 현재 화면, 관심종목, 보유종목, 미체결 주문 종목을 우선 구독한다.
- 동일 종목·채널의 중복 구독을 제거한다.
- 메시지를 시장 공통 이벤트로 정규화해 UI, 차트 집계기, 체결 엔진에 배포한다.
- 구독별 마지막 거래소 시각과 수신 시각으로 `live/delayed/stale/offline` 상태를 계산한다.
- 재연결 뒤 REST 스냅샷으로 현재 상태를 보정한다.

### FR-03 차트

- REST로 과거 분봉·일봉을 채운 뒤 WebSocket 체결로 진행 중인 봉을 갱신한다.
- 1분·5분·15분·30분·60분·일·주 봉을 MVP에서 제공한다. 틱 차트는 후속 단계다.
- 각 봉의 OHLC, 거래량, 거래대금을 제공하고 거래량·거래대금을 별도 보조 패널로 전환한다.
- 가격·거래량·거래대금 기준 SMA/EMA 기간을 사용자가 설정하고 interval별 로컬 설정으로 복원한다.
- 로컬 모의 fill의 매수·매도, 부분·전량 체결 가격을 시장 candle과 분리된 marker layer로 표시한다.
- 거래소 시각 기준으로 봉을 만들며 표시 시 KST/거래소 시간 전환을 제공한다.
- 중복 체결, 역순 도착, 세션 경계, 휴장일을 처리한다.
- 수정주가 여부와 데이터 범위를 표시한다.

### FR-03A 지수·선물 시장 컨텍스트

- KOSPI·KOSDAQ·KOSPI200 현물지수와 KOSPI200 주간·KRX 야간 선물을 별도 상품·세션으로 표시한다.
- Nasdaq 상장 주식·ETF, Nasdaq 현물지수, NQ/MNQ 선물을 서로 다른 instrument로 관리한다.
- WTI CL/MCL 등 해외선물은 실제 월물, 만기, 거래소, 통화, 승수, tick size를 표시한다.
- 무료 기본안은 QQQ·IWM·USO를 Nasdaq·Russell 2000·WTI 방향의 ETF 프록시로 제공하고 `PROXY_LIVE`를 표시한다.
- KIS WebSocket과 계정 권한이 확인된 채널만 `live`로 표시한다.
- REST polling 지수, 유료 시세 미신청, 장 마감, 미지원 채널을 각각 `delayed/restricted/closed/unsupported`로 구분한다.
- 선물 최근월과 연속선물을 구분하고 롤오버·세션·DST 정보를 숨기지 않는다.
- 자세한 공급자/권한/세션 계약은 `docs/13-realtime-market-coverage.md`를 따른다.

### FR-04 순위와 스캐너

- 상승률, 하락률, 거래량, 거래대금, 시가총액, 체결강도 순위를 제공한다.
- 국내/미국, 거래소, 업종, 가격대, ETF 포함 여부로 필터링한다.
- KIS 원본 순위와 로컬 계산 순위를 구분한다.
- 거래량/거래대금 증가율의 기준을 화면과 저장 데이터에 명시한다.
  - `전일 동시간 대비`: `(오늘 누적 - 전일 동시간 누적) / 전일 동시간 누적`
  - `전일 최종 대비`: `(오늘 누적 - 전일 최종) / 전일 최종`
- 분모가 0이거나 기준 데이터가 없으면 값을 만들지 않고 `N/A`와 사유를 표시한다.
- 계층형 산업·테마 매핑으로 거래대금 가속도, 시장 점유율, 상승 종목 breadth, 상위 종목 집중도를 계산해 오늘의 주도주·주도 테마를 표시한다.
- 반도체→소부장처럼 상·하위 테마를 함께 표시하되 이중합산을 막고 대형주 한 종목 독주와 광범위 테마 강세를 구분한다.

### FR-05 종목 상세

- 차트와 호가창은 동일 종목의 한 페이지에서 항상 동시에 보이며 탭·별도 창·collapse로 둘 중 하나를 숨기지 않는다.
- 코드, 이름, 시장, 거래소, 통화, 업종, 상품 유형을 표시한다.
- 현재가, 전일 대비, 등락률, OHLC, 거래량, 거래대금, 시가총액, 52주 고저를 표시한다.
- 국내는 지원 범위의 다단계 호가, 미국은 KIS가 제공하는 호가 깊이만 표시한다.
- API가 제공하지 않는 값은 추정으로 채우지 않는다.

### FR-06 뉴스와 시황

- 국내, 미국, 관심종목, 보유종목으로 분류한다.
- 제목, 제공사, 발행 시각, 관련 종목, 원문 식별자/링크, 수집 시각을 저장한다.
- URL 또는 공급자 식별자와 정규화 제목으로 중복을 묶는다.
- 본문 사용권이 확인되지 않으면 제목과 링크만 노출한다.
- 미국 뉴스가 KIS 제공 범위에서 검증되지 않으면 기능을 “제한됨”으로 표시하고 공급자 어댑터를 비워 둔다.

### FR-07 움직임 설명

- 가격 변화, 거래량·거래대금 급증, 지수·업종 대비 초과수익, 갭/고점 돌파, 수급, 호가 불균형, 관련 뉴스를 근거로 사용한다.
- 결과는 한 줄 요약, 수치 근거, 관련 뉴스, 반대 근거, 신뢰도, 기준 시각을 포함한다.
- KOSPI·KOSDAQ과 미국 소형 급등주는 공식 촉매, 시장구조 추정, 희석·거래정지 위험을 별도 카드로 표시한다.
- 근거가 부족하면 `확인 불가` 또는 `낮은 신뢰도`로 반환한다.
- LLM은 선택 기능이며 저장된 근거만 요약한다. 새로운 사실이나 출처를 만들 수 없다.

### FR-07A 공시 감시

- 미국은 SEC EDGAR, 국내는 OpenDART에서 신규 공시를 근실시간 감시한다.
- 공급자가 WebSocket을 제공하지 않으면 이용 정책 범위의 증분 polling/RSS를 사용한다.
- 인수·합병·자산취득/매각·영업양수도·분할·증자·자사주·지배권 변경을 canonical event로 분류한다.
- 신규 공시는 번역과 분석을 기다리지 않고 먼저 UI와 explanation evidence에 전달한다.
- 미국 공시는 원문을 보존하고 제목·핵심 요약을 한국어로 번역한다.
- 번역 상태, 원문 링크, 접수 시각, 감지 시각, 번역 완료 시각을 표시한다.
- 번역이나 자동 분류가 실패해도 원문 공시 알림은 유실하지 않는다.

### FR-07B 장외 선행 보조지표

- Hyperliquid XYZ의 SMSN·SKHX·HYUNDAI와 CL 등 공개 HIP-3 시세를 선택적으로 수집한다.
- 온체인 TradFi perp를 KRX 현물, CME 선물 또는 확정 예측으로 표시하지 않는다.
- mark/oracle basis, funding, OI, 거래대금, impact spread와 세션을 함께 표시한다.
- SKHX 보통주와 SKHY ADS의 계약 단위를 구분하고 큰 괴리에서는 자동 판단을 중단한다.
- 유동성·freshness·metadata 검증을 통과하지 못하면 `NO_SIGNAL`을 반환한다.
- walk-forward 검증 전에는 상승확률 대신 장외 매수·매도 우위로 표시한다.

### FR-07C 글로벌 사건과 시장 영향

- 중앙은행·금리, 물가·고용·성장·PMI, 국채·달러·유동성, 재정·세금·관세·수출통제, 선거·정책·규제, 은행·신용위기, 공급망·파업, 원유·가스·금속·농산물, 전쟁·제재·해상 사고, 재난·감염병·사이버, 주요 기업 실적·산업 충격·AI 기술 경쟁을 공통 사건으로 정규화한다.
- 경제지표는 actual, consensus, prior, revised prior, 단위, 발표시각과 원 발표기관을 보존하고 예상 대비 surprise를 exact decimal로 계산한다.
- 전체 category coverage registry에 `AVAILABLE/UNSUPPORTED/MISSING/DELAYED/STALE`를 기록해 무료 기본안에서 알 수 없는 영역을 숨기지 않는다.
- KIS 해외뉴스 제목은 빠른 탐지 신호로 사용하고 SEC, Nasdaq Trader, 기업 IR, 정부·규제·해상기관 등 공식 1차 자료로 교차 확인한다.
- Reuters는 데이터 라이선스 식별자가 설정된 경우에만 사용하며 무료 수집·본문 스크래핑 대상으로 삼지 않는다.
- 미국·영문 사건은 원문과 게시 시각을 보존하고 한국어 번역·요약 상태를 표시한다.
- 사건 전후 KOSPI·KOSDAQ, USD/KRW, QQQ·USO proxy, 관련 미국 종목과 국내 업종의 가격·거래량·거래대금 반응을 비교한다.
- 경제적 전달 경로, 실제 반응, 대안 가설·반대 근거, freshness와 confidence를 분리하며 상관관계를 확정 인과로 표현하지 않는다.

### FR-07D 국내·미국 시장 이벤트 캘린더

- 실적 발표, 배당, ex-dividend, 유상증자, 무상증자, 액면분할·병합, 자사주, M&A, IPO, lock-up 해제, 거래정지·규제 이벤트를 국내·미국 공통 캘린더 이벤트로 정규화한다.
- FOMC, CPI, PPI, PCE, GDP, 고용, PMI, 금리결정, 옵션만기, 선물만기, MSCI·Russell·ETF 리밸런싱과 주요 경제지표를 같은 캘린더에 표시한다.
- 전체 캘린더 페이지는 국내·미국·글로벌 이벤트를 모두 보여주고, 날짜 선택 시 해당 날짜의 이벤트를 발표 시각 순서로 정리한다.
- 종목 workspace에서는 선택 종목의 시장에 맞는 이벤트를 우선 표시한다. 국내 종목은 국내 및 글로벌 이벤트, 미국 종목은 미국 및 글로벌 이벤트를 표시한다.
- 직접 관련 종목 이벤트는 `instrumentIds`, 시장 전체 이벤트는 `affectedMarkets`로 필터링한다.
- 이벤트에는 provider, sourceEventId, evidence ID, 공개·수신·감지 시각, timezone, data quality를 포함한다.
- `HEADLINE_ONLY`, `DELAYED`, `STALE`, `UNSUPPORTED` 상태를 숨기지 않고 UI와 도메인에 전파한다.
- actual/consensus/prior/revision은 exact decimal string으로 저장하며, consensus 권리가 없으면 surprise를 계산하지 않는다.
- 자세한 계약과 provider 범위는 `docs/21-market-event-calendar.md`를 따른다.

### FR-08 모의 주문

- 주문 방향: 매수/매도
- 주문 유형: 시장가/지정가
- 수량: 국내·미국 모두 MVP에서는 정수
- 상태: `pending`, `accepted`, `partially_filled`, `filled`, `cancelled`, `rejected`
- 제출 전 장 상태, 호가 단위, 수량, 통화별 가용 현금, 가용 보유량, 데이터 신선도를 검증한다.
- `clientOrderId`를 멱등키로 사용해 중복 제출을 막는다.
- 시장가는 실제 visible 호가를 가격 순서대로 소비하고 depth 부족 잔량을 로컬 취소한다.
- 지정가는 주문 이후 실제 체결가가 지정가에 도달하거나 통과한 관측 수량 안에서 부분체결한다.
- 고급 모델은 선행 대기수량 추정, 실제 체결량 차감, VI·상하한가·동시호가·장전후 세션을 명시적 품질과 함께 처리한다.

### FR-09 가상 계좌와 손익

- KRW와 USD 현금을 별도로 보관하고 암시적으로 합산하지 않는다.
- 환전은 명시적인 가상 환전 원장 거래로만 수행한다.
- 현금은 가용, 주문 예약, 결제 대기로 구분한다.
- 평균단가법을 MVP 기본값으로 사용하고 계산식을 테스트로 고정한다.
- 실현손익, 미실현손익, 수수료, 세금, 환차손익을 분리한다.
- 원장 기록은 불변이며 잔고와 포지션을 원장에서 재구축할 수 있어야 한다.

### FR-10 설정과 진단

- 초기 자본, 통화, 수수료·세금·슬리피지, 데이터 stale 기준을 설정한다.
- 토큰 값이나 비밀키를 제외한 REST 호출 수, 오류, 재시도, WebSocket 구독 수를 본다.
- DB 백업·복원과 무결성 검사를 제공한다.
- 수수료·세금·거래 시간은 시행일 기반 규칙으로 관리하며 값 변경에 앱 배포가 필수이지 않게 한다.

## 5. 비기능 요구사항

### 성능

- WebSocket 수신부터 UI 반영까지 정상 부하 중앙값 300ms 이내
- 호가·체결 폭주 시 50~100ms 배치 렌더링으로 renderer 멈춤 방지
- 캐시가 있는 순위 첫 화면 3초 이내
- 종목 워크스페이스 핵심 시세 2초 이내, 나머지 패널은 점진 로딩

### 안정성

- DB 쓰기는 트랜잭션이며 WAL 모드를 사용한다.
- 앱 강제 종료 후 원장 무결성이 유지된다.
- 재연결, 중복 메시지, 중복 주문에 멱등성을 갖는다.
- 마이그레이션 전 자동 백업한다.

### 정확성

- 금액은 부동소수점으로 저장하지 않는다. 통화별 최소 단위 정수 또는 정확한 decimal 표현을 사용한다.
- 거래소 시각은 IANA timezone과 UTC instant를 함께 보존한다.
- 원본 API payload fixture와 정규화 결과의 계약 테스트를 둔다.
- 오래된 가격으로 산출한 평가액에는 stale 표시를 전파한다.

## 6. MVP 수용 기준

- 종목을 열면 REST 스냅샷 뒤 WebSocket 체결이 현재가와 진행 중인 봉을 함께 갱신한다.
- WebSocket 강제 단절 시 상태가 바뀌고 자동 재연결되며, 이전 값에는 stale 표시가 붙는다.
- 국내/미국별 상승·하락·거래량·거래대금 목록과 기준·통화·갱신 시각이 보인다.
- 현금 1,000만 원 계좌에서 매수 체결 후 주문, 체결, 현금 원장, 포지션이 한 트랜잭션으로 일치한다.
- 보유량 초과 매도와 가용 현금 초과 매수는 거부된다.
- 지정가는 반대 호가가 조건을 만족할 때만 체결 후보가 된다.
- 앱 재시작 후 현금, 포지션, 주문, 체결 내역이 동일하다.
- 움직임 설명은 최소 한 개의 수치/뉴스 근거를 표시하며, 없으면 확인 불가라고 말한다.
- 비밀키·토큰이 renderer, 로그, 평문 DB에 나타나지 않는다.
- 통합 테스트에서 KIS 주문 URL 호출은 0회다.
