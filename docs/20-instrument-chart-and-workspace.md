# 종목 차트와 동일 페이지 작업공간

## 1. 불변 UI

종목 워크스페이스에서는 해당 종목의 **차트와 호가창을 항상 같은 페이지에 동시에 표시**한다. 탭 전환, 모달, 별도 창을 거쳐야 둘 중 하나가 보이는 구성은 허용하지 않는다. 패널 크기는 조절할 수 있지만 차트와 호가창의 `display:none`, collapse, unmount는 허용하지 않는다.

좁은 화면에서는 주문·뉴스·설명 패널을 아래로 이동한다. 차트와 호가창은 최소 너비를 지키며 같은 viewport에 남는다. 최소 지원 해상도에서 둘을 동시에 제공할 수 없으면 앱이 축소 레이아웃을 쓰고 수평 스크롤로 숨기지 않는다.

## 2. 차트 데이터

- KIS REST 과거봉으로 초기화하고 KIS WebSocket 체결로 진행 봉을 갱신한다.
- OHLC, 거래량, 거래대금을 각 봉에 보존하고 0과 `UNAVAILABLE/null`을 구분한다.
- 1분·5분·15분·30분·60분·4시간·일·주 봉을 계약에 포함한다.
- 분봉 계열은 당일 범위, 일·주봉은 6개월·1년·5년 범위를 제공하고 지원하지 않는 주기/범위 조합은 IPC 경계에서 거부한다.
- 장전, 정규장, 동시호가, 장후 구간을 세션 배경으로 구분한다.
- 수정주가 여부, 통화, 원천, 거래소 시각, freshness를 표시한다.
- 거래량과 거래대금은 서로 다른 하단 패널이며 0과 미제공을 구분한다.

거래량·거래대금에는 `PROVIDER_REPORTED/LOCAL_TRADE_AGGREGATE/UNAVAILABLE` provenance를 붙인다. 거래대금이 공급자에서 직접 오면 그 값을 사용한다. 로컬 계산이면 `SUM_TRADE_PRICE_TIMES_QUANTITY`, exact-decimal, output scale, 반올림 규칙을 기록하고 공급자 원본처럼 표시하지 않는다.

완료 봉은 `closedAt <= asOf`여야 한다. 진행 봉은 `FORMING`으로 명시하고 `openedAt <= asOf < closedAt`인 현재 interval 하나만 허용한다.

## 3. 이동평균선

사용자는 가격 종가, 거래량, 거래대금을 기준으로 SMA 또는 EMA를 설정할 수 있다. 기간은 2~500, 전체 선은 최대 12개로 제한한다. 기본 가격 이평선은 5·20·60·120이며 색상·표시 여부는 사용자 설정으로 로컬 저장한다.

중복 설정은 거부하고 데이터가 기간보다 짧으면 선을 앞쪽으로 억지 연장하지 않는다. 분봉과 일봉의 설정은 interval별로 보존한다.

## 4. 모의 매수·매도 마커

차트 마커는 로컬 SQLite의 실제 모의 `fill`만 사용한다. 주문 제출 가격과 체결 가격을 혼동하지 않는다.

- 매수/매도, 부분/전량 체결을 모양과 텍스트로 함께 구분한다.
- 마커에는 체결가, 체결수량, 체결시각, 로컬 주문 ID를 연결한다.
- 동일 주문이 여러 가격에서 부분체결되면 각 fill을 개별 마커로 남긴다.
- 겹친 마커는 묶어서 표시하되 펼치면 원래 fill을 모두 확인할 수 있어야 한다.
- 취소·거부 주문은 가격 체결 마커가 아니며 주문 타임라인에서만 표시한다.

시장 캔들의 원천은 `KIS_CANONICAL_MARKET_DATA`, 마커의 원천은 `LOCAL_PAPER_FILL`로 고정한다. 로컬 마커가 KIS나 실제 증권계좌의 주문으로 보이게 만들지 않는다.

## 5. 호가와 차트 동기화

종목을 전환하면 같은 subscription generation으로 호가·체결·차트·주문 티켓을 함께 교체한다. 이전 종목 이벤트는 새 화면에 반영하지 않는다. 차트 crosshair 시각에 맞춰 해당 시점의 체결과 로컬 fill을 조회할 수 있지만, 과거 호가 replay가 없으면 현재 호가를 과거 호가처럼 표시하지 않는다.

국내 KIS 호가는 현재 검증된 10매도·10매수 단계를 표시한다. 미국 KIS 호가는 현재 검증된 1매도·1매수만 표시하며 10단계로 보간하지 않는다.

## 6. 계약과 테스트

- `src/contracts/instrument-chart.ts`: candle, indicator, paper-fill marker, 동일 페이지 visibility 계약
- `tests/instrument-chart-contract.test.ts`: OHLC, 종목 일치, 중복 이평선, 차트·호가 동시 표시 회귀 테스트
- E2E: 종목 전환 중 데이터 혼입 없음, 부분체결 마커, 설정 복원, 최소 해상도 동시 표시

## 7. 현재 연결 상태 (2026-07-20)

Electron renderer는 KIS 키나 REST client를 소유하지 않는다. `charts.getHistory` preload API가 신뢰된 main frame의 요청만 Node sidecar로 전달하고, sidecar가 KIS 읽기 전용 REST를 호출해 검증된 projection만 renderer에 보낸다.

- `1m`: KIS 당일 분봉을 최대 2 page(60개) 조회한다. 장 마감 뒤 요청 cursor는 15:30으로 제한해 장후 데이터가 정규장 history page를 밀어내지 않게 한다. 15:20~15:29 예상체결 carry-forward 행은 제외하고 15:30 확정 후보를 15:29~15:30 최종 봉으로 정규화한다. 개별 분 거래대금은 공급자가 주지 않으므로 `UNAVAILABLE`이다.
- `1d`: 6개월·1년·5년 버튼을 제공한다. KST 달력 기준 시작일을 계산하고 각각 최대 2·4·15 page를 역방향 조회해 공급자 거래량·거래대금을 보존한다.
- 모의투자 REST 호출 제한을 고려해 차트 pagination은 초당 1회 이하로 직렬화하고, 5년 요청을 위해 worker timeout은 120초로 제한한다.
- 진행 중인 마지막 실제 봉은 신선한 KIS WebSocket 현재가로 high/low/close를 갱신한다.
- `5m/15m/30m/60m/4h`: 현재 연결한 KIS `inquire-time-itemchartprice` 당일 1분봉을 최대 14 page까지 조회해 KRX 세션 시작점 기준으로 묶는다. 4시간 정규장 봉은 09:00~13:00과 13:00~15:20으로 잘리고 마감 동시호가 확정 봉은 별도 세션 봉이다. 마지막 경계 page에 포함된 이전 영업일 행은 current-day 계약에서 제외한다. 누락된 원본 봉을 생성하지 않으며 분봉 거래대금 `UNAVAILABLE`을 전파한다.
- KIS 공식 샘플의 `inquire-time-dailychartprice` 과거 분봉 endpoint는 별도 adapter가 아직 연결되지 않았다. 따라서 현재 제품에서 4시간봉은 당일 범위이며 과거 4시간봉을 지원한다고 표시하지 않는다.
- `1w`: 선택한 6개월·1년·5년 KIS 수정 일봉을 월요일 09:00~금요일 15:30 KST 경계로 집계하고 공급자 거래량·거래대금을 합산한다.
- 같은 interval의 범위를 빠르게 바꿀 때 renderer는 `interval:range` 요청 키로 늦게 도착한 과거 응답을 무시한다.
- 실차트가 `READY`가 아니면 합성 차트를 KIS 데이터로 라벨링하지 않는다.

5년 일봉 원본은 최대 2,000봉까지 projection으로 보존하지만 renderer는 최대 360개 시각 버킷으로 결정론적 downsample한다. 각 버킷은 첫 open, 최고 high, 최저 low, 마지막 close와 합산 거래량·거래대금을 보존하며 화면에 원본/표시 개수를 명시한다. SMA/EMA는 downsample 전 원본 일봉으로 계산한 뒤 각 버킷 끝점에 투영한다. 1,250봉 fixture 회귀 테스트가 렌더 상한과 exact integer 거래량 합산을 검증한다.
