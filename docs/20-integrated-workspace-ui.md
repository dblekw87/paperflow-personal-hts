# PaperFlow 통합 투자 작업공간 UI

## 목적

PaperFlow의 전체 화면과 기능 흐름은 관심종목, 시장 순위, 종목 상세,
차트·호가, 뉴스·공시·상승 요인, 로컬 모의계좌가 하나의 선택 종목을
공유하는 데스크톱 작업공간으로 구성한다.

참고 화면의 정보 구조와 밀도는 활용하되 타사 로고, 고유 문구, 아이콘,
색 배치, 컴포넌트 외형을 복제하지 않는다.

## 전체 구조

1. 왼쪽 레일: 주요 기능 전환
2. 왼쪽 패널: 사용자가 별표로 명시적으로 추가한 관심종목
3. 상단 스트립: 시장 상태, 국내·미국 지수, 선물·환율·원자재
4. 중앙 작업영역: 순위 또는 선택 종목의 차트·10호가
5. 오른쪽 컨텍스트 패널: 종목 요약, 상승·하락 이유, 뉴스, 공시
6. 하단 스트립: 연결 상태, 현재 데이터 출처, 로컬 계좌 상태

## 종목 선택 동작

- 순위 행 또는 관심종목을 누르면 `activeInstrumentId`를 한 번만 변경한다.
- 검색 결과와 순위 행 선택은 화면 이동만 수행하며 관심종목을 자동 추가하지 않는다.
- 관심종목 추가·삭제는 종목 헤더의 별표 이벤트로만 수행하고 로컬에 보존한다.
- 차트, 호가, 헤더, 보유 수량, 뉴스·공시, 상승·하락 요인이 같은 종목으로
  함께 전환되어야 한다.
- 이전 종목의 늦은 REST·WebSocket 응답은 새 종목 화면에 반영하지 않는다.
- 차트와 호가창은 종목 상세 화면에서 항상 동시에 보여야 한다.

## 시장 순위

- KRX 범위를 명시한다.
- 현재가·마지막 종가, 등락률, 조회 거래일 현재 거래량, 전일 총거래량 대비
  장중 누적 거래량 증감률, 조회 거래일 현재 거래대금을 보여준다.
- 거래량·거래대금은 거래일마다 새로 시작하는 KIS 당일 값만 사용한다.
- 상승은 코랄, 하락은 블루, 보합은 중립색으로 표시한다.
- 회사 아이콘은 사용하지 않는다.
- 1~3위 순위 배지, 고정 헤더, 행 호버·키보드 포커스를 제공한다.
- `평균거래량` API 후보군을 전체 KRX 오늘 거래량 순위라고 표현하지 않는다.

## 종목 헤더

- 현재가, 전일 대비, 시가, 고가, 저가, 오늘 거래량, 오늘 거래대금을 표시한다.
- 정규장 종료 후 최종 체결가는 영업일 날짜가 확인되지 않으면
  `마지막 종가`로 표시한다.
- 수신 시각과 KRX/NXT/통합 여부를 가격과 분리해 명시한다.

## 호가창

- KIS REST 최종 10호가로 먼저 채운 뒤 WebSocket 프레임으로 갱신한다.
- 모든 호가 행의 왼쪽 슬롯은 매도, 오른쪽 슬롯은 매수로 고정한다.
- 주문 방향을 행 위치나 매도·매수 호가 영역에서 추론하지 않는다.
- 슬롯 클릭 시 공통 수량 입력값과 해당 행 가격으로 로컬 지정가 주문을 만든다.
- KIS 실시간 체결의 `CTTR`을 체결강도로 표시하며, 미수신 시 `—`로 둔다.
- 매도 10호가는 왼쪽에 옅은 블루 잔량 깊이 막대, 매수 10호가는 오른쪽에
  옅은 코랄 잔량 깊이 막대를 최대 잔량 대비 비율로 표시한다.
- 중앙에는 각 호가와 전일 종가 대비 개별 등락률을 함께 표시한다.
- 상단 우측에는 52주 고·저, 상·하한가, VI, 시가·고가·저가, 거래량,
  전일 거래량 대비, 중간호가 기준정보를 배치한다. 제공자가 주지 않은 값은
  추정하지 않고 `—`로 표시한다.
- 하단 왼쪽에는 관측한 실제 누적 거래량의 증가분으로만 구성한 최근 체결가·
  체결량 테이프를 표시한다. 첫 프레임 또는 누적량 역행 시 체결량을 만들지 않는다.
- 매수 지정가가 최우선 매도호가보다 낮으면 `미체결 대기`이며 예약 현금만
  반영한다. 체결 전에는 보유수량과 평가손익이 변하지 않는다.
- 장 종료 또는 연결 중단 시 마지막 실제 호가 배열을 그대로 유지한다.
- 장전 첫 조회처럼 실제 잔량이 없고 저장된 실호가도 없는 경우에는 KIS 종목
  마스터에서 `STOCK`으로 확인된 종목만 현재가 기준 가격 단위를 표시하고 잔량은
  `미수신`으로 둔다. ETF·ETN·미확인 상품에는 주식 호가 단위를 적용하지 않으며,
  이 가격 안내는 주문·체결 근거가 아니다.
- 마지막 수신 호가는 `장마감` 또는 `멈춤` 상태와 수신 시각을 표시한다.
- stale 호가를 실시간으로 승격하지 않는다.
- stale·장마감 호가의 클릭 주문은 차단한다.
- 주문은 로컬 SQLite 모의계좌에만 기록한다.

## 차트

- 가격 캔들과 하단 거래 보조영역을 한 차트 안에 둔다.
- 하단은 거래량과 거래대금을 별도 패널로 나누지 않는다.
- 거래량은 넓은 방향색 막대, 거래대금은 같은 x 위치의 좁은 오버레이 막대로
  표시한다.
- 두 막대는 서로 다른 단위와 독립 정규화 축을 사용함을 범례에 명시한다.
- 툴팁에는 거래량과 거래대금을 함께 표시한다.
- KIS가 봉별 거래대금을 제공하지 않는 1분봉은 거래대금을 만들거나
  추정하지 않고 미제공으로 표시한다.

## 오른쪽 컨텍스트 패널

- KIS 뉴스 제목, SEC, DART, 공식 기관·거래소·기업 자료를 시간순으로 합친다.
- 미국 자료는 원문 링크와 한국어 번역을 함께 보관한다.
- 상승·하락 이유는 관찰된 시점, 근거, 반대 근거, 신뢰도와 함께 표시한다.
- 상관관계를 확정 인과로 표현하지 않는다.

## 시장 이벤트 캘린더

- 전체 캘린더 페이지는 국내·미국·글로벌 이벤트를 같은 월간 grid에 표시한다.
- 전체 캘린더 페이지와 날짜 agenda, 하단 예정 이벤트 목록은 현재 선택 종목과
  무관하게 모든 기업·시장 일정을 표시한다.
- 월간 날짜 셀에는 긴 제목을 넣지 않고 중요도 dot, event kind icon, 개수만 표시한다.
- 날짜를 선택하면 오른쪽 agenda에서 해당 날짜 이벤트를 발표 시각 순으로 보여준다.
- agenda 카드는 시간, 시장, 중요도, 이벤트 종류, 종목, provider, data quality,
  actual/consensus/prior를 표시한다.
- 필터는 국내, 미국, 글로벌, 기업, 거시, 파생, 리밸런싱, 중요도, 공식/집계/지연
  상태를 제공한다.
- 별도 종목 workspace 캘린더 패널을 만들 경우에는 선택 종목 시장을 따른다. `KRX:*`/`NXT:*`는
  국내 및 글로벌 이벤트, `NASDAQ:*`/`NYSE:*`/`AMEX:*`/`NYSEARCA:*`는 미국 및
  글로벌 이벤트를 기본 표시한다.
- 직접 관련 종목 이벤트는 `instrumentIds`가 선택 종목과 일치할 때 우선 표시하고,
  시장 전체 이벤트는 `affectedMarkets`가 선택 종목 시장과 일치할 때 표시한다.
- 캘린더 projection이 로딩 중이면 월간 셀 dot, agenda 카드, 종목 범위 목록에
  Skeleton UI를 표시하고 합성 이벤트 제목을 만들지 않는다.

## 거래 화면 로딩 게이트

- Electron 실제 데이터 모드의 종목 거래 화면은 선택 종목의 차트 projection이
  현재 interval/range와 일치하고 candle을 하나 이상 포함할 때까지 전체 카드
  표면을 Skeleton UI로 유지한다.
- 호가, 뉴스, 시장지수, 순위, 투자자 동향이 먼저 도착해도 차트가 준비되기 전에는
  해당 값을 부분 노출하지 않는다.
- fixture preview 모드는 이 게이트를 적용하지 않는다.
- `HEADLINE_ONLY`, `DELAYED`, `STALE`, `UNSUPPORTED`는 카드와 상세 drawer에서
  숨기지 않는다.
- 차트의 이벤트 marker는 발표 시점 표식일 뿐 매수·매도 신호로 보이지 않게 한다.

## 품질 기준

- 실제값과 fixture를 같은 화면에서 섞지 않는다.
- 데이터 출처, 시장 범위, 현재/지연/마감 상태를 항상 노출한다.
- 테이블 숫자는 tabular 숫자와 열 고정 폭을 사용한다.
- 14px 기본 글꼴, 다크·라이트 테마, 키보드 탐색을 유지한다.
- 타사 주문 API 호출은 금지하고 KIS는 읽기 전용 허용 목록만 사용한다.
